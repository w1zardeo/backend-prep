# Day 08: TypeScript Strictness та Валідація Даних

## 1. Теорія (Warm-up Quiz)
- **Map vs Object:** Map кращий для частого додавання/видалення ключів та зберігає порядок.
- **HTTP 409 Conflict:** Використано правильно для запобігання дублікатів (email).
- **Backpressure:** Розуміння того, як не переповнити RAM при роботі з потоками.

## 2. Практика: TypeScript Integration
- **tsconfig.json:** Встановлено `strict: true`. Це фундамент, який забороняє `any` та змушує обробляти `null/undefined`.
- **Interface vs Runtime:** Реалізовано `parseJsonBody` з перевіркою типів під час виконання.

## 3. Reflection
- **TSC Check:** Команда `npx tsc --noEmit` пройшла успішно.
- **"Aha!" Moment:** TypeScript допоміг виявити потенційну помилку при зверненні до властивостей об'єкта, який міг бути `undefined` після пошуку в Map.
